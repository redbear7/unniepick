## 1. Summary

언니픽 앱에 현재 브론즈/실버/골드 3단계 레벨을 10단계 세분화 등급제로 개편하고, 사용자가 가게 메뉴 가격을 직접 제보하면 포인트·배지·혜택으로 보상하는 시스템을 도입한다. 이를 통해 사용자의 앱 재방문율과 콘텐츠 품질(가격 정확도)을 동시에 높인다.

## 3. Background

### 지금 무슨 상황인가?

언니픽은 창원 지역 뷰티·식음료 가맹점의 쿠폰과 혜택을 모아주는 앱이다. 지금은 브론즈 / 실버 / 골드, 3가지 레벨만 있다. 레벨이 3개뿐이라 사용자가 빨리 골드에 도달하고 더 이상 성장하는 느낌을 받지 못한다. 앱에 오래 머물 이유가 약하다.

최근 **가격 제보 기능(PriceReportScreen)** 이 개발 완료되었다. 사용자가 직접 가게 메뉴 가격을 입력할 수 있다. 그러나 현재 제보에 대한 **보상이 없다.** 보상 없이는 제보 참여율이 낮아 데이터 품질이 나빠진다.

### 왜 지금인가?

- 가격 제보 기능이 막 출시되어 사용자 보상 설계가 시급하다.
- Supabase `profiles` 테이블에 `level`, `level_progress`, `point_to_next` 컬럼이 이미 준비되어 있다. 즉, DB 인프라는 갖춰진 상태다.
- 초기 사용자를 락인(lock-in)할 최적의 시점이다. 초기에 성장 루프를 만들어야 나중에 MAU를 높이기 쉽다.

## 4. Objective

### 목표

> **"언니픽 사용자가 앱을 꾸준히 쓸 이유를 만든다."**

레벨 시스템과 보상이 있으면 사용자는 "오늘 뭔가 하면 내 레벨이 오른다"는 동기를 갖는다. 가격 제보 보상은 앱의 콘텐츠(가격 정보)를 사용자가 직접 채워주도록 유도한다.

### 회사에게 주는 이익

- 재방문율 상승 → 광고·쿠폰 노출 기회 증가
- 가격 데이터 품질 향상 → 가게 입점 영업 시 신뢰 자료로 활용
- 커뮤니티 느낌 강화 → 경쟁 앱 대비 차별화

### 사용자에게 주는 이익

- 활동할수록 더 좋은 쿠폰·혜택을 받는다
- 내 레벨과 배지를 보며 성취감을 느낀다

### Key Results (OKR 형식)

| # | 목표 지표 | 측정 기준 | 목표값 (출시 후 3개월) |
|---|-----------|-----------|----------------------|
| KR1 | 월 활성 사용자(MAU) 재방문율 | 2주 내 재방문 비율 | 40% 이상 |
| KR2 | 가격 제보 건수 | 월간 제보 수 | 월 200건 이상 |
| KR3 | 레벨 3 이상 달성 사용자 비율 | 전체 가입자 대비 | 30% 이상 |
| KR4 | 투표(맞아요/틀려요) 참여율 | 제보 1건당 평균 투표 수 | 제보 1건 당 5표 이상 |

## 5. Market Segment

### 누구를 위해 만드는가?

| 세그먼트 | 설명 | 주요 니즈 |
|----------|------|-----------|
| **일상 소비자** | 창원에서 카페·맛집·뷰티샵을 자주 이용하는 20~40대 여성 | "더 싸게, 더 좋은 혜택으로 쓰고 싶다" |
| **정보 제공자** | 가게를 자주 다니며 실제 가격을 아는 사람 | "내가 아는 정보가 도움이 됐으면 좋겠다" |
| **성취 추구자** | 게임·앱에서 레벨업이나 배지 모으기를 좋아하는 사람 | "뭔가 쌓이는 느낌이 좋다" |

### 제약 조건

- 현재 서비스 지역: **창원시** (마산·진해 포함) 한정
- 1인 개발 → 복잡한 기능은 단계적으로 출시해야 함
- 포인트 현금화(출금)는 법적 이슈(상품권업 등록 등)로 **이번 범위 외**

## 6. Value Proposition

### 사용자가 해결하고 싶은 일(Jobs to be done)

1. **"이 가게 진짜 가격이 얼마인지 알고 싶다"** → 직접 제보하면 실제 가격 정보가 쌓인다
2. **"쿠폰만 받고 끝인 앱은 재미없다"** → 레벨·배지·보상으로 게임 같은 재미 추가
3. **"나는 앱을 많이 쓰는데 혜택이 더 있어야 한다"** → 고레벨 전용 쿠폰·배지로 충성도 보상

### 사용자가 얻는 것

- 레벨이 오를수록 쿠폰 할인율·개수 혜택 증가
- 가격 제보 시 즉시 포인트 획득
- 투표(맞아요/틀려요)로도 소소한 포인트 적립
- 특별 배지(제보왕, 인증왕 등)로 자랑거리 생김
- 생일 쿠폰, 주년 쿠폰 등 레벨별 추가 혜택

### 경쟁 앱 대비 차별점

| 비교 항목 | 일반 쿠폰 앱 | 언니픽 |
|-----------|-------------|--------|
| 가격 정보 | 가게가 직접 입력 | 사용자+가게 공동 입력 → 신뢰도↑ |
| 레벨 시스템 | 없거나 단순 3단계 | 10단계 세분화 + 배지 |
| 지역 특화 | 전국 단위 | 창원 집중 → 밀착감↑ |

## 7. Solution

### 7.1 사용자 흐름 (User Flow)

```
앱 사용 행동 (쿠폰 다운, 제보, 투표 등)
        ↓
  포인트 적립 + 경험치(XP) 누적
        ↓
  레벨 바(level_progress) 채워짐
        ↓
  레벨업 시 → 축하 애니메이션 + 배지 지급 + 혜택 해금
        ↓
  마이페이지에서 내 등급·배지·혜택 확인
```

#### 7.2.3 가격 제보 보상 상세

가격 제보는 신뢰도 있는 데이터를 만들기 위해 **2단계 보상 구조**를 사용한다.

**1단계 — 제보 즉시 (임시 포인트)**
- 제보 후 바로 "임시 포인트 +30P" 지급 (마이페이지에 '검토 중' 표시)

**2단계 — 승인 후 (확정 포인트 + XP)**
- 관리자가 제보를 승인하면 → 확정 포인트 +30P, XP +30 지급
- 거절 시 → 임시 포인트 취소 (사유 알림 발송)

**추가 보상 조건**

| 조건 | 추가 보상 |
|------|-----------|
| 제보한 가격에 '맞아요' 10표 이상 | 보너스 XP +20 |
| 한 달 내 제보 20건 이상 | '이달의 제보왕' 배지 + XP +200 |
| 처음 제보한 메뉴인 경우 | '퍼스트 제보' 배지 + XP +50 |

### 7.3 기술 스펙

#### DB (Supabase)

**`profiles` 테이블 — 기존 컬럼 활용**
```
level          integer   -- 1~10
level_progress integer   -- 현재 레벨 내 진행 XP
point_to_next  integer   -- 다음 레벨까지 남은 XP
```

**신규 추가 컬럼 (profiles)**
```sql
total_xp       integer   DEFAULT 0,   -- 전체 누적 XP
point_balance  integer   DEFAULT 0,   -- 현재 포인트 잔액
badge_ids      text[]    DEFAULT '{}' -- 획득한 배지 ID 목록
```

**신규 테이블: `xp_logs`**
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
user_id     uuid REFERENCES profiles(id),
action      text,           -- 'price_report', 'coupon_use', 'vote', ...
xp_amount   integer,
point_amount integer,
created_at  timestamptz DEFAULT now()
```

**신규 테이블: `price_reports`**
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
user_id     uuid REFERENCES profiles(id),
store_id    uuid REFERENCES stores(id),
menu_name   text,
price       integer,
status      text DEFAULT 'pending',  -- 'pending' | 'approved' | 'rejected'
reject_reason text,
created_at  timestamptz DEFAULT now()
```

#### 앱 (React Native / Expo)

- **레벨업 애니메이션**: Lottie 또는 Reanimated 2로 레벨업 시 팡파레 효과
- **마이페이지**: 레벨 바, 배지 그리드, 포인트 잔액, XP 로그 리스트
- **가격 제보 화면 (기존 PriceReportScreen)**: 제보 후 보상 피드백 모달 추가

### 7.4 가정 (Assumptions)

| # | 가정 | 검증 방법 |
|---|------|-----------|
| A1 | 사용자는 레벨업 알림에 반응하여 앱을 재방문할 것이다 | 푸시 클릭률 측정 |
| A2 | 포인트 사용처(쿠폰 슬롯 등)가 매력적으로 느껴질 것이다 | 포인트 소진율 추적 |
| A3 | 제보 승인률이 70% 이상일 것이다 (허위 제보 최소화) | 어드민 승인/거절 비율 모니터링 |
| A4 | 레벨 5 이상 달성 시 이탈률이 낮아질 것이다 | 코호트 분석 |
| A5 | 10레벨이 너무 어렵지 않고 6개월 내 달성 가능한 난이도다 | XP 시뮬레이션 후 조정 |

## 8. Release

### Phase 1 — 핵심 MVP (약 3~4주)

> 레벨 시스템 기반 구축 + 가격 제보 보상 연동

- [ ] DB 스키마 추가 (`xp_logs`, `price_reports` 상태 업데이트, profiles 컬럼 추가)
- [ ] XP 적립 로직 (Supabase Edge Function 또는 앱 내 처리)
- [ ] 레벨 계산 함수 (`total_xp` → `level`, `level_progress`, `point_to_next`)
- [ ] 마이페이지: 레벨 바 + 포인트 잔액 표시
- [ ] 가격 제보 승인 시 XP + 포인트 지급 (어드민 승인 액션)
- [ ] 레벨업 푸시 알림

### Phase 2 — 배지 + 투표 보상 (약 2~3주)

- [ ] 배지 지급 로직 (조건 달성 시 자동 지급)
- [ ] 마이페이지 배지 그리드 UI
- [ ] 투표 참여 XP 반영
- [ ] 제보왕 배지 자동 판정 (월말 집계)

### Phase 3 — 포인트 사용처 + 레벨업 애니메이션 (약 2~3주)

- [ ] 포인트 리딤 기능 (쿠폰 슬롯, 랜덤 뽑기 등)
- [ ] 레벨업 축하 애니메이션
- [ ] XP 로그 히스토리 화면
- [ ] 6개월 비활동 강등 자동화 (Cron)

### Phase 4 — 고도화 (출시 후 피드백 반영)

- [ ] 레벨 난이도 조정 (XP 기준값 튜닝)
- [ ] 시즌 이벤트 (기간 한정 배지, XP 2배 이벤트)
- [ ] 친구 초대 보상 강화
- [ ] 등급별 전용 쿠폰 큐레이션 자동화
