# PRD — 언니픽 (UnniePick)
**위치기반 쿠폰 피드 & 빠른알림 플랫폼**

> 버전 1.0 · 2026-04-16 · 작성자: bangju

---

## 1. Summary

언니픽은 동네 가게 사장님이 쿠폰을 발행하고, 손님이 한 화면에서 발견·저장·사용까지 마치는 **위치기반 쿠폰 피드 앱**이다.  
SNS 팔로우 구조(Twitter/X)와 일본 최대 쿠폰북 핫페퍼(HotPepper)의 쿠폰 편의성을 결합해, **채팅만큼 익숙한 UX**로 동네 소비를 연결한다.  
사장님에게는 팔로워에게 즉시 쿠폰을 보내는 "원클릭 알림" 채널을, 손님에게는 저장한 가게의 혜택을 놓치지 않는 개인 쿠폰함을 제공한다.

## 3. Background

### 3-1. 지금 무슨 문제가 있나?

동네 가게 사장님은 쿠폰을 만들어도 손님에게 전달할 수단이 없다.  
- 네이버 블로그·인스타그램에 올리면 **알고리즘에 묻힌다**  
- 카카오채널은 메시지 비용이 높고 설정이 복잡하다  
- 단골 손님은 "오늘 뭐 할인해?" 를 알 방법이 없다  

손님 입장에서도 쿠폰이 흩어져 있다.  
- 네이버·카카오·배달앱 등 **앱마다 따로 저장**해야 한다  
- 쿠폰 만료일을 놓쳐서 혜택을 못 쓴다  
- 내 주변 가게의 오늘 혜택을 **한 눈에 볼 공간이 없다**  

### 3-2. 왜 지금인가?

- 스마트폰 보급률 99%, 20-30대는 앱 전환 비용이 낮다  
- 일본 핫페퍼는 지역 쿠폰 O2O로 연 수천억 엔 매출을 증명했다  
- 국내에 **위치기반 + SNS 팔로우 + 쿠폰**을 동시에 잡은 서비스가 없다  
- Expo + Supabase 스택으로 1인 개발자도 MVP를 빠르게 출시할 수 있다  

### 3-3. 핵심 인사이트

> 쿠폰은 할인 정보가 아니라 **사장님의 타임라인 포스트**다.  
> 손님은 팔로우한 가게의 포스트(쿠폰·공지)를 피드에서 읽고, 마음에 들면 저장하고, 가게에서 꺼낸다.

## 4. Objective

### 4-1. 목표

**사장님이 30초 안에 쿠폰을 발행하고, 손님이 3탭 안에 사용할 수 있게 한다.**  

- 사장님 가치: "단골에게 오늘의 혜택을 즉시 알릴 수 있는 나만의 채널"  
- 손님 가치: "팔로우한 가게 혜택을 매일 확인하는 개인 쿠폰 피드"  
- 회사 가치: 구독 수익 + 데이터 기반 지역 소비 인사이트  

### 4-2. 측정 지표 (OKR)

| # | Objective | Key Result | 기준 시점 |
|---|-----------|------------|----------|
| O1 | 사장님이 쿠폰 발행을 반복한다 | 가게당 월 평균 쿠폰 발행 수 ≥ 4건 | 출시 3개월 |
| O2 | 손님이 쿠폰을 실제로 사용한다 | 저장 쿠폰 대비 사용률 ≥ 30% | 출시 3개월 |
| O3 | 팔로우 관계가 늘어난다 | 가게당 평균 팔로워 ≥ 50명 | 출시 6개월 |
| O4 | 수익이 발생한다 | 유료 구독 사장님 ≥ 100명 | 출시 6개월 |
| O5 | 손님이 앱에 자주 돌아온다 | 월간 활성 사용자(MAU) 대비 주간 리텐션(WAU) ≥ 40% | 출시 6개월 |

## 5. Market Segment

### 5-1. 사용자 A — "단골을 챙기고 싶은 동네 사장님"

| 항목 | 내용 |
|------|------|
| 직업 | 카페·음식점·미용실·네일샵·편의점 등 소상공인 |
| 고민 | 단골에게 오늘 세일을 알릴 방법이 없다 |
| 기존 행동 | 인스타 스토리·카카오채널·종이 쿠폰 |
| 불편함 | 시간이 오래 걸리고, 도달률을 알 수 없다 |
| 원하는 것 | "지금 바로 단골한테 쿠폰 쏘고 싶다" |

### 5-2. 사용자 B — "할인 혜택을 놓치기 싫은 20-30대 손님"

| 항목 | 내용 |
|------|------|
| 세대 | 20-30대, 스마트폰 네이티브 |
| 고민 | 자주 가는 가게의 혜택을 놓친다 |
| 기존 행동 | 인스타 팔로우, 앱 알림 허용, 배달앱 쿠폰 |
| 불편함 | 쿠폰이 여러 앱에 흩어져 있고, 만료일을 잊는다 |
| 원하는 것 | "팔로우한 가게 혜택이 하나의 피드로 보였으면" |

### 5-3. 제약 조건

- MVP는 iOS 전용 (Expo)  
- 초기 서비스 지역: 특정 1-2개 도시 (수도권 집중)  
- 결제 없음: MVP에서는 쿠폰 무료 발행, 수익은 구독으로만

### 7-2. Key Features

#### Feature 1 — 쿠폰 발행 (사장님)

| 항목 | 내용 |
|------|------|
| 유형 | 할인율 (%), 금액 할인 (원), 무료 증정 |
| 입력 필드 | 쿠폰 제목, 대상 메뉴/서비스, 할인 조건, 수량 한도, 기간 |
| 발행 후 | 팔로워 전체에 즉시 푸시 알림 (Expo Notifications) |
| 재알림 | 발행 후 원하는 시점에 "재알림 보내기" 1탭 |
| 수정/삭제 | 발행 중인 쿠폰 수정 가능 (단, 이미 저장한 사용자는 업데이트 내용 고지) |

#### Feature 2 — 쿠폰 저장 & 사용 (손님)

| 항목 | 내용 |
|------|------|
| 저장 방법 | 피드에서 "쿠폰 받기" 탭 1회 |
| 사용 방법 | 내 쿠폰함 → QR 사용하기 → 전체화면 QR |
| 만료 알림 | D-1, D-0 자동 푸시 |
| 상태 관리 | 사용 가능 / 사용 완료 / 만료 자동 분류 |

#### Feature 3 — 팔로우 시스템

| 항목 | 내용 |
|------|------|
| 팔로우 | 가게 탐색 또는 가게 프로필에서 팔로우 |
| 언팔로우 | 피드에서 가게 카드 롱탭 또는 프로필에서 |
| 팔로우 시 | 해당 가게의 모든 신규 쿠폰 푸시 알림 자동 수신 |
| 팔로워 수 | 사장님 대시보드에 실시간 표시 |

#### Feature 4 — 위치기반 탐색

| 항목 | 내용 |
|------|------|
| 기본 | GPS 기반 현재 위치 자동 감지 |
| 반경 필터 | 300m / 500m / 1km / 3km 선택 |
| 카테고리 | 카페, 음식, 미용, 편의점, 기타 |
| 정렬 | 거리순, 쿠폰 많은 순, 팔로워 많은 순 |

#### Feature 5 — 알림 시스템

| 알림 종류 | 트리거 | 수신자 |
|-----------|--------|--------|
| 신규 쿠폰 | 사장님 쿠폰 발행 | 해당 가게 팔로워 |
| 쿠폰 만료 D-1 | 자동 (자정 배치) | 쿠폰 저장한 손님 |
| 쿠폰 만료 D-0 | 자동 (자정 배치) | 쿠폰 저장한 손님 |
| 재알림 | 사장님 수동 발송 | 아직 사용 안 한 팔로워 |
| 사용 확인 | QR 스캔 완료 | 사장님 앱 (진동+소리) |

#### Feature 6 — 사장님 대시보드

| 지표 | 설명 |
|------|------|
| 팔로워 수 | 현재 팔로워 총수, 주간 증감 |
| 발행 쿠폰 수 | 활성 / 종료 분리 |
| 저장 수 | 쿠폰별 저장한 손님 수 |
| 사용 수 | 실제 사용된 횟수, 사용률 (%) |
| 도달률 | 알림 발송 수 대비 쿠폰 저장 전환율 |

## Appendix — 핵심 화면 플로우

```
[손님 온보딩]
앱 설치 → 소셜 로그인 → 위치 권한 허용
→ 관심 카테고리 선택 (카페/음식/미용)
→ 주변 가게 추천 5개 → 팔로우 선택
→ 홈 피드 진입 (첫 쿠폰 노출)

[쿠폰 사용 플로우]
홈 피드 쿠폰 카드 발견
→ "쿠폰 받기" 탭 (저장)
→ 가게 방문
→ 내 쿠폰함 → QR 사용하기
→ 전체화면 QR 표시
→ 사장님 스캔
→ "사용 완료" 처리

[사장님 쿠폰 발행 플로우]
사장님 탭 → "쿠폰 발행하기"
→ 유형 선택 → 내용 입력 → 미리보기 확인
→ "팔로워에게 즉시 알림" 탭
→ 팔로워 전원 푸시 발송
→ 대시보드에서 저장/사용 실시간 확인
```
