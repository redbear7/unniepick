01

문제 정의
& 가설

매장 월드컵은 이미 있다. 하지만 결과가 어디에도 흘러가지 않는다. 사용자에겐 순위 차트로만 보이고, 사장님은 자기 매장이 몇 위인지도 모른다. 양쪽 가치가 버려지는 중.

Problem
"열심히 뽑은 투표가
어디로도 안 간다."
사용자 월드컵 플레이는 주 1.4만 건 수준이지만, 이 데이터가 매장·딜·광고 어느 축에도 연결되지 않는다. 사장님은 "우리 가게가 뽑혔는지" 모르고, 사용자는 "내가 뽑은 곳이 혜택이 있는지" 모른다. 참여 루프가 소비되지 않고 휘발.
Hypothesis
"투표를 딜로 바꾸면
광고 매출 30%↑."
IF 월드컵 결과를 매장 리포트·프리미엄 슬롯·무료 크레딧 루프로 연결한다면, THEN 사장님 유료 전환율 +15%pt, 언니픽 광고 매출 3개월 내 +30%, 사용자 월드컵 참여율 2.1배 (내 투표 = 실제 변화로 인식).
Why now · Yelp Ads 반성

Yelp Ads는 CPC 기반으로 동네 소상공인 진입 장벽이 월 $150+. 결과적으로 동네 개인 카페·네일샵은 사실상 광고 불가능. 언니픽은 이미 사용자가 자발적으로 참여하는 컨텐츠(월드컵)를 가졌고, 결과를 그대로 광고 상품으로 전환할 수 있다 — 마케팅 예산 대신 투표 점수가 노출의 자격이 됨.

03

솔루션
아키텍처

세 개의 루프 — Vote (사용자) · Report (사장님) · Monetize (수익). 투표 데이터 한 번 수집하면 세 경로로 동시에 흐른다.

Loop 01
Vote
— 투표 캡쳐
M1

카테고리 × 동네 세분 월드컵

기존 전체 월드컵을 카테고리 × 동네 단위로 쪼갠다. 예: 상남동 카페 월드컵, 양덕동 네일샵 월드컵. 한 번에 16강 — 동네 소상공인의 참여 가능성이 극적으로 상승.

M2

주간 시즌 시스템

매주 월요일 00:00 새 시즌 시작, 일요일 23:59 결산. 매장별 승·패·총투표수 누적. 투표 결과는 다음 주 월요일 오전 리포트로 사장님께 발송.

M3

투표자 보상 — '픽이 실제가 됐어요'

내가 상위에 투표한 매장이 프리미엄 슬롯에 등장하면 '내 픽이 1등!' 뱃지 + 전용 쿠폰 자동 발급. 사용자 입장에선 투표가 게임 → 실제 혜택 루프로 진화.

Loop 02
Report
— 주간 리포트
M4

사장님 앱 홈 · 주간 Scoreboard

매주 월요일 사장님 앱 홈 상단에 "지난주 성적표" 카드 노출. 카테고리 순위·총 투표수·전주 대비 변화·다음 시즌 상대 매장 미리보기. 숫자 3개만 보여주는 간결함.

M5

이메일 · 카톡 리포트 (옵션 구독)

앱 외부 채널로도 리포트 발송. 월드컵에 참여도 안 한 사장님에게 "당신 매장이 카페 월드컵 32위에 진입"을 Push — 앱 재활성화 트리거.

M6

투표 인사이트 (무료) — 투표자 페르소나

누가 당신 매장에 투표했나 — 20대 여성 67%, 주거리 반경 1.5km, 시간대 야간 집중 등 익명 집계 인사이트. PII 절대 비노출. Yelp Analytics의 무료 버전.

Loop 03
Monetize
— 수익 전환
M7

상위 매장 · 무료 프리미엄 슬롯 1주 증정

카테고리 월드컵 우승 매장은 다음 주 앱 홈 상단 프리미엄 슬롯 1주 무료. 광고 예산 없이도 '투표로 이기면 노출'이라는 신뢰 루프 확립 — Yelp Ads의 정반대.

M8

유료 — 결승 고정 참여권 (₩30K/주)

다음 주 월드컵 '결승 8강 풀'에 자동 진출 보장. 투표 트래픽을 사는 것 — 기존 CPC보다 예측 가능한 상품. 소상공인에게도 명확한 가격.

M9

프리미엄 — 카테고리 스폰서십 (월 단위)

"상남동 카페 월드컵 Presented by ○○" 형태. 월 ₩150K. 사용자 경험 훼손 없이(상단 노출 + 우승자 증정 이벤트 주최) 브랜드 인지 획득. Phase 2 이후 확장.

05

시스템
의존성

기존 매장 월드컵 기능이 대부분의 인프라를 커버. 신규는 시즌·리포트·수익 상품 3가지.

D1
기존 월드컵 엔진 · 매칭·투표 파이프라인 재사용

현재 동작 중인 월드컵 엔진에 카테고리·동네 필터 파라미터만 추가. 투표 데이터 스키마도 동일 테이블 확장.

D2
시즌 스케줄러 · cron + Redis 신규

매주 월요일 00:00 새 시즌 시작, 일요일 23:59 결산. Redis로 active season id 관리. 리포트 생성은 월요일 07:00 배치.

D3
리포트 렌더링 엔진 · 서버 생성 이미지 신규

이메일·카톡 리포트용 이미지 렌더링 (Puppeteer + HTML 템플릿). 앱 내 Scoreboard는 JSON API로 실시간.

D4
인앱 결제 · 프리미엄 슬롯·결승 고정 신규

사장님 앱 내 ₩ 결제 (PG 토스·NICE). 자동 영수증·세금계산서 발행. 구독이 아닌 주 단위 선결제 방식.

D5
광고 슬롯 매니저 · 기존 홈 배너 확장 확장

앱 홈 상단의 기존 배너 슬롯에 "월드컵 우승" 라벨 붙인 프리미엄 슬롯 추가. Live Activity 타깃팅과 연동.

D6
알림 채널 · 카톡 비즈·이메일 발송 신규

앱 외부 사장님(앱 재활성화 타깃)에게 카톡 알림톡·이메일로 발송. 템플릿은 카톡 비즈 심사 필요 (사전 4주).

데이터 모델
제안

신규 4개 — worldcup_season · worldcup_match · worldcup_vote · merchant_ad_slot. 기존 매장 월드컵 테이블은 season_id만 추가.

-- 주간 시즌 · 카테고리 × 동네 단위 CREATE TABLE worldcup_season ( id BIGSERIAL PRIMARY KEY, category_key VARCHAR(20) -- 'cafe' | 'food' | 'beauty' ..., region_key VARCHAR(20) -- '상남동' | '양덕동' ..., started_at TIMESTAMPTZ NOT NULL, ends_at TIMESTAMPTZ NOT NULL, status VARCHAR(12) -- 'active' | 'closed' | 'archived', UNIQUE(category_key, region_key, started_at) ); CREATE INDEX ix_ws_active ON worldcup_season(status) WHERE status = 'active';
-- 투표 로그 · 성과 리포트 base CREATE TABLE worldcup_vote ( id BIGSERIAL PRIMARY KEY, season_id BIGINT NOT NULL, match_id BIGINT NOT NULL, user_id BIGINT NOT NULL, winner_mid BIGINT NOT NULL, loser_mid BIGINT NOT NULL, voted_at TIMESTAMPTZ DEFAULT now(), user_region VARCHAR(20), user_age INT -- 집계용, 저장은 bucket, user_gender VARCHAR(2) ); CREATE INDEX ix_wv_season ON worldcup_vote(season_id, winner_mid); -- 리포트 생성 쿼리 최적화용

리스크
& 완화

광고 상품화의 가장 큰 위험은 사용자 경험 훼손. 투표가 '돈으로 산 결과'처럼 보이는 순간 루프 붕괴.

!
Risk 01 · Integrity
투표 조작 의심 — 월드컵 결과 신뢰성 하락
유료 슬롯을 산 매장이 '우승'한 것처럼 보이면 사용자가 월드컵 자체를 불신.
완화
프리미엄 슬롯과 월드컵 결과 UI를 시각적으로 완전 분리. "광고" 라벨 명시. 월드컵 랭킹은 투표 수 그대로.
!
Risk 02 · Vote Manipulation
사장님이 지인 동원해 투표 조작
친척·알바 계정으로 대량 투표. 우승 → 무료 슬롯 수령으로 부정 이득.
완화
사용자당 시즌 동일 매장 10표 상한. IP·디바이스 ID 동일 패턴 탐지. 의심 시즌은 결과 재검증 프로세스.
!
Risk 03 · Small Merchant Exclusion
큰 가게만 계속 우승 → 소상공인 격차 고착
유명 카페가 매주 우승 → 신규·작은 매장은 노출 기회 없음.
완화
우승은 직전 4주 연속 우승 시 1주 쿨다운. 또한 '신규 매장 피켓' 월드컵 별도 트랙 운영.
!
Risk 04 · Report Fatigue
주간 리포트 푸시가 스팸처럼 느껴짐
매주 월요일 발송이 사장님에게 피로. 3주 차 오픈율 급락 가능.
완화
첫 번째 리포트는 강렬 (순위·투표수 큼직). 2번째부터는 전주 대비 변화가 클 때만 푸시. 외부 채널은 구독 토글.
!
Risk 05 · Monetize Over-stretch
수익화 우선 → 월드컵 재미 감소
결승 고정 유료 상품이 많아지면 사용자 매치에 '광고 매장'만 보여 흥미 감소.
완화
결승 고정은 8강 풀의 최대 25%(2석)만 허용. 나머지는 투표수 순. 유료 슬롯과 오가닉 슬롯 비율 모니터링 KPI.

오픈 퀘스천

스펙 확정 전 PM·비즈·디자인이 함께 답할 질문들.

?
Q1 · Monetize
결승 고정 가격 ₩30K/주가 적정한가?
너무 낮으면 매출 의미 없고, 너무 높으면 소상공인 진입 불가. Yelp Ads($150/월 $ = ₩35K/주) 벤치마크와 유사.
결정 기준
P2에 ₩30K로 시작. 구매 전환율 5% 미달이면 인하, 10% 초과면 인상 A/B.
?
Q2 · UX
사용자에게 '내 픽 보상'을 얼마나 강조할 것인가?
강조하면 참여율 ↑ 하지만 "혜택 사냥꾼"만 남을 위험. 약하면 루프 가동 안됨.
결정 기준
투표 중엔 숨기고, 결과 발표 후에만 뱃지·쿠폰 서프라이즈. Instagram Stories 스타일.
?
Q3 · Business
매장 월드컵 자체에 투표하지 않은 매장 → 제외할 것인가?
현재 월드컵 엔진은 매장 자체 등록이 불필요. 하지만 '경쟁 상대가 올라왔는데 본인은 모름' 시나리오 가능.
결정 기준
전 매장 자동 참여 (opt-out만). opt-out 매장도 리포트는 받음 — 참여 유도.
?
Q4 · Legal
매장 '우승' 타이틀 광고 사용은 허용되는가?
사장님이 자기 매장 외부에 "언니픽 상남동 카페 월드컵 우승"을 표기하고 싶어할 수 있음. 법적·브랜드 가이드 필요.
결정 기준
공식 배지 이미지 제공 + 사용 가이드(글꼴·로고 병기 등). 매장 외부 사용은 환영.
