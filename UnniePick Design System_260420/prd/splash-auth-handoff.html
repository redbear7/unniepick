01

왜 이
형태인가

스플래시 6종 · 회원가입 basic/enhanced 2종 중 Phase 1 런칭용 결정. 근거는 이탈률 · 구현 비용 · 법적 요구사항.

결정 01 — 스플래시

V1 Centered 채택

화이트 배경 + 오렌지 워드마크 + 하단 태그라인 + v1.0.0 + 1800ms. 6종 중 가장 표준적. 캐릭터/심볼 variant는 브랜딩 확정 전 보류.

결정 02 — 회원가입 톤

Basic variant 채택

Enhanced의 🐻 HelperMsg는 친근하지만 scope 크리프 리스크. Phase 1은 깔끔한 iOS 표준으로 런칭, Phase 2에서 A/B.

결정 03 — 6스텝 구조

SMS + 프로필 + GPS + 팔로우 + 푸시

휴대폰 인증 = 법적 본인 확인, 팔로우 5곳 = Day-1 리텐션 seed. GPS·푸시는 맥락적으로 요청 (GPS 직전 = 팔로우 화면 직전, 푸시 = 가입 완료 후). 둘 다 풀스크린 설명.

03

세부
결정사항

UI Kit 프로토타입을 프로덕션으로 옮길 때 명확히 해야 할 정책. 개발자와 PM이 미리 합의.

스플래시 지속시간
고정 1800ms
세션 검증이 빠르게 끝나도 최소 1800ms 유지 (브랜드 각인). 2초 이상 걸리면 끝나는 시점까지 hold 연장. 5초 초과 시 hold 상태에서 "다시 시도" 표시.
SMS 제공자
Supabase Auth + NHN 톡플러스
Supabase Phone Auth 기본 사용, SMS 게이트웨이는 NHN 톡플러스 SMS (KR 요구사항). 데모에서는 아무 6자리 통과하지만 프로덕션은 실제 OTP 검증.
닉네임 정책
한글 5자 = 영문 8자 (가중치 1.6)
한글이 포함되면 한글 문자수 기준 /5, 순수 영문/숫자는 /8 표시. 금칙어: admin, 욕설, "언니"/"사장" 등 서비스 용어 · 실제 상호명.
팔로우 최소 5곳
하드 강제 · 스킵 불가
Day-1 리텐션의 관건. 4곳 이하일 때 CTA 비활성. 반경 500m 내 가게 < 10일 경우 반경 자동 확장 (1km → 2km). "건너뛰기" 링크는 의도적으로 미구현.
약관 필수 2종
이용약관 · 개인정보 (법적)
마케팅 동의는 선택. 필수 중 하나라도 미체크면 "동의하고 시작하기" 비활성. 약관 원문은 /terms/tos, /terms/privacy 정적 페이지. 동의 타임스탬프 user_consents 테이블 기록.
GPS 권한
풀스크린 pre-prompt · 거부 시 진행 불가
Step 4에서 설명 스크린 후 iOS CLLocationManager.requestWhenInUseAuthorization() 호출. 거부 시 다시 요청 CTA + 설정 앱 딥링크 안내. 권한 없이는 반경 탐색 · 자동 매장 감지 불가 = 서비스 코어가치 차단.
푸시 알림 권한
풀스크린 pre-prompt · 거부해도 진행
Step 6에서 가치 설명 후 Notification.requestPermission(). 거부 · 나중에 → 홈 진입 허용. APNs 토큰 발급 성공 시 user_push_tokens에 등록. MY 탭 설정에서 언제든 다시 활성화 가능.
뒤로가기 정책
Step 1에서는 앱 종료
Step 2-6에서 뒤로가기 시 이전 스텝. Step 1에서 뒤로가기 시 iOS 표준: 앱 홈화면으로. Step 2에서 번호 수정 시 입력된 6자리 OTP 폐기. Step 4/6(권한) 뒤로가기 = 이전 스텝으로 (권한 거부가 아닔).

엣지

SMS 수신 실패

OTP 5회 오류
해당 번호 30분 잠금. 에러 다이얼로그 + Step 1으로 복귀. 서버 auth_attempts 기록.

Step 2-4 진행 중 백그라운드 → foreground 시 state 보존. 30분 이상 방치 시 Step 1부터 재시작.

진행 불가 — "다시 요청" CTA + 설정 앱 딥링크 배너. 설정에서 허용 후 앱 복귀 시 자동 재체크.
푸시 권한 거부 · Step 6
거부해도 홈 진입 허용. user_push_tokens에 null 기록. MY > 설정에서 언제든 다시 활성화 가능. 만료 임박 쿠폰 알림 명시적 CTA 노출.
반경 내 가게 < 10
500m → 1km → 2km 순서로 반경 확장. 창원 전체에서도 10개 미만이면 있는 만큼 표시 + 최소 선택 기준 조정 (ceil(n/2)).
닉네임 서버 검증 실패
클라에서 통과한 닉네임이 서버 금칙어 업데이트로 거부될 수 있음. 프로필 저장 시점에 재검증 → Step 3 에러 복귀.
약관 버전 업데이트
앱 최초 실행이 아니어도 약관 해시가 바뀌면 홈 진입 전 약관 시트 재노출. 필수 2종만 재확인.
네트워크 단절
OTP 발송 · 검증 · 프로필 저장 각 호출 실패 시 retry CTA. Step 4 팔로우 커밋은 idempotent (user_id + store_id unique).
약관 동의 없이 닫기
바텀시트 외부 탭 · 뒤로가기 시 동의 상태 초기화. Step 4 본문은 유지. 다시 완료 탭 시 빈 체크박스로 시트 재노출.
