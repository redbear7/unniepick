02

스펙 간
연결 구조

세 스펙은 독립 배포되지만, 공통 인프라(GPS·지갑·Live Activity)를 공유한다. 먼저 배포된 스펙의 데이터가 뒷 스펙의 품질을 끌어올리는 누적 구조.

Spec 01 → Spec 02
쿠폰 사용 = 스탬프 자동 적립
Idea 01의 '사용완료' 이벤트가 Idea 02의 스탬프 적립 트리거. 사용자는 한 번의 결제로 두 루프에 진입.
Spec 01 → Spec 03
우승 매장 노출 우선순위 +1
Idea 03 월드컵 우승 매장은 Idea 01 Live Activity 타깃팅에서 우선순위 가중치. 유료 슬롯 가치 증명.
Spec 02 → Spec 03
단골 시크릿 쿠폰 → 투표 보상
Idea 02 단골 해금 시크릿 쿠폰을 Idea 03의 '내 픽이 1등' 보상으로도 옵션 활용. 매장 자율 선택.
공통 인프라 · GPS
매장 좌표 DB · PostGIS
Idea 01에서 구축한 매장 좌표 인프라를 Idea 02·03 모두 재사용. 사장님 '내 가게 위치 조정' 기능도 공통.
공통 인프라 · Live Activity
ActivityKit · APNs liveness
Idea 01의 Live Activity 인프라에 Idea 02 단골 배지·Idea 03 우승 알림 등 state를 얹는 구조.
공통 인프라 · 지갑
coupons 테이블 재사용
세 스펙 모두 기존 coupons 테이블을 확장 (is_secret 컬럼 추가 등). 지갑 렌더링 파이프라인 단일.
