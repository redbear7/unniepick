01

문제 정의
& 가설

언니픽은 '첫 방문'은 유도하지만 '재방문'을 이끄는 장치가 없다. 사장님 입장에서 '우리 단골' 이라는 식별자도 존재하지 않는다.

Problem
"두 번째 방문할
이유가 없어요."
언니픽을 통한 첫 방문 후 동일 매장 재방문율 19% (내부 가정치). 쿠폰은 '새 손님 유치' 목적이라 기존 손님에게는 혜택이 없고, 사장님도 '우리 단골'을 시스템상 인식할 방법이 없다. 결과적으로 쿠폰앱 → 단골앱으로 진화하지 못한다.
Hypothesis
"적립 × 등급 루프로
재방문 2.5배."
IF 매장별 스탬프 10개로 단골 등급을 해금하고 비공개 혜택(시크릿 쿠폰·히든 메뉴)을 부여한다면, THEN 재방문율 19% → 48%, 쿠폰앱의 정체성이 '할인앱' → '내 동네 관계' 로 전환되어 M3 리텐션 +14%pt.
Why now

Starbucks Rewards·스타벅스 리워드처럼 대형 브랜드의 적립 루프는 이미 표준이지만 동네 소상공인은 아날로그 스탬프 카드(종이)에 머물러 있다. 언니픽은 이미 매장 DB·결제 흐름·지갑 구조를 갖추고 있어 — 디지털 스탬프로 이식하는 마찰이 가장 낮다.

03

솔루션
아키텍처

세 개의 레이어 — 적립(Earn) · 등급(Tier) · 해금(Unlock). 각 레이어는 독립 배포 가능하고, 기존 지갑 구조와 나란히 공존.

Layer 01
Earn
— 스탬프 적립
M1

매장 QR 스캔 · 1일 1회 제한

매장 테이블·카운터에 붙은 고유 QR 코드를 앱 내 '스캔' 버튼으로 찍는다. 같은 매장은 24h 쿨다운으로 재스캔 차단 — 악용 방지. 스캔 즉시 도장 애니메이션 + haptic success.

M2

영수증 OCR 크로스 체크 (옵션)

의심 스캔(GPS 불일치·비정상 빈도)에 한해 영수증 업로드 요청 가능. Phase 2에서 전면 확대 검토. MVP는 QR only.

M3

GPS Fence — 스캔 시 좌표 검증

스캔 순간 기기 GPS ↔ 매장 좌표 거리 50m 이내만 허용. 캡쳐·공유 악용 차단. Idea 01의 좌표 인프라 재사용.

Layer 02
Tier
— 단골 등급
M4

매장별 단골 등급 · 3단

첫방문(1장) · 자주감(5장) · 단골(10장). 매장 단위로 독립 계산 — A카페 단골이라고 B카페 단골은 아니다. 등급 달성 시 전체 화면 축하 애니메이션 + Live Activity 등급업 배지.

M5

등급 감쇠 (Decay) — 90일 미방문

90일간 재방문 없으면 한 단계 하락. 소멸 예고 D-7 Push로 복귀 유인. Starbucks Rewards 방식 차용.

M6

시티 티어 (글로벌 등급)

매장별 단골 합산으로 '상남동 단골왕' 도시 레벨 등급 별도 계산. 초기엔 비노출, Phase 2에서 리더보드·배지로 활용.

Layer 03
Unlock
— 해금 혜택
M7

시크릿 쿠폰 — 일반 쿠폰보다 50% 더 깊이

단골만 보이는 쿠폰. 지갑 탭에서 자물쇠 해제 연출로 언락. 매장 사장님이 직접 설정 (사장님 앱).

M8

히든 메뉴 · 비공개 서비스

메뉴판에 없는 '단골 전용' 메뉴·서비스. 예: 네일샵 '단골 전용 케어 5분 추가', 카페 '숨은 시그니처 라떼'. 매장별 자율.

M9

우선 예약·줄서기 인식

사장님 앱 대시보드에 "단골 ○○님이 입장하셨어요" 실시간 알림. 네일샵·미용실 예약 대기시 자동 우선권.

악용 시나리오 6종 & 대응

E1 · QR 캡쳐 후 집에서 스캔

→ GPS fence 50m 불일치로 거부. status=out_of_range

E2 · 친구가 QR 사진 공유

→ 동일 매장 동일 계정 24h 쿨다운으로 재적립 차단.

E3 · QR 사진 인쇄 매장 외부 게시

→ QR에 5분 회전 nonce 포함 (사장님 앱이 주기 갱신하는 동적 QR). 정적 캡쳐 무효화.

E4 · 같은 매장 여러 계정 로테이션

→ 디바이스 ID·IP 패턴 탐지 · 신규 계정 첫 스캔은 결제 검증 요청 트리거.

E5 · 매장 직원이 손님 몰래 스캔

→ 스캔은 반드시 사용자 계정에서 발생 — 사장님이 대신 적립 불가한 구조.

E6 · 매장 옆 가게에서 스캔

→ GPS fence 50m 내에서만 허용. 혼잡 지역은 사장님이 반경 축소 요청 가능.

05

시스템
의존성

기존 언니픽 구조를 최대한 재사용한다. Idea 01의 GPS·Live Activity 인프라를 그대로 활용하면 개발 마찰이 크게 감소.

D1
사장님 앱 · 동적 QR 발급 모듈 신규

5분 주기로 nonce 회전하는 QR 이미지를 사장님 앱 '내 가게 > 스탬프 QR' 화면에 노출. 프린트 A4 템플릿도 함께 제공 (4주마다 재인쇄 안내).

D2
QR 스캐너 · AVFoundation iOS 내장

별도 SDK 불필요. AVCaptureMetadataOutput + qr 타입만 쓰면 됨. 카메라 권한 First Use 시 요청.

D3
GPS geofence · Idea 01 공용 인프라 재사용

Idea 01에서 이미 구축한 매장 좌표 DB와 PostGIS ST_DWithin 쿼리를 그대로 재사용. 신규 코드는 스캔 시점 검증 로직만.

D4
Live Activity · 등급업 배지 업데이트 재사용

Idea 01이 구축한 Live Activity 인프라에 stamp badge state 추가. 단골 등급 달성 시 push-to-start로 "단골 승급" 배지 3분 노출.

D5
애니메이션 · Lottie or SwiftUI native 디자인 협업

도장 애니메이션은 Lottie 또는 SwiftUI matchedGeometryEffect 로 구현. 잉크 번짐·황금 도장은 SVG 마스킹. 디자이너가 원화 14 스프라이트 납품 필요.

D6
Haptic · UIImpactFeedbackGenerator iOS 내장

스캔 성공 light · 도장 찍힘 medium · 등급업 heavy + success notification. 모든 감각 피드백은 무음 환경에서도 작동.

데이터 모델
제안

신규 테이블 3개 — stamp_ledger, stamp_tier, unlockable_benefit. 기존 coupons 테이블은 is_secret 플래그만 추가.

-- 스탬프 원장 · 적립 단위 레코드 CREATE TABLE stamp_ledger ( id BIGSERIAL PRIMARY KEY, user_id BIGINT NOT NULL, merchant_id BIGINT NOT NULL, qr_nonce VARCHAR(32) NOT NULL, scanned_at TIMESTAMPTZ DEFAULT now(), gps_point GEOGRAPHY(POINT), gps_delta_m INT, source VARCHAR(12) -- 'qr' | 'receipt' | 'admin' ); CREATE UNIQUE INDEX ux_sl_cooldown ON stamp_ledger(user_id, merchant_id, date(scanned_at)); -- 같은 날 · 같은 매장 1회만
-- 매장별 단골 등급 · 사용자 × 매장 매트릭스 CREATE TABLE stamp_tier ( user_id BIGINT, merchant_id BIGINT, stamp_count INT DEFAULT 0, tier VARCHAR(16) -- 'new' | 'regular' | 'vip', tier_since TIMESTAMPTZ, last_visit_at TIMESTAMPTZ, decay_warn_at TIMESTAMPTZ, PRIMARY KEY (user_id, merchant_id) ); CREATE INDEX ix_st_decay ON stamp_tier(last_visit_at) WHERE tier != 'new'; -- 90일 미방문 스캔용

API 엔드포인트
제안

앱 쪽 2개, 사장님 앱 2개. 동적 QR은 사장님 앱이 주기적으로 nonce를 갱신하는 형태로 유지.

/v1/stamps/scan
사용자가 QR 스캔 후 호출. 서버가 nonce·매장·GPS·24h 쿨다운 전부 검증. 성공 시 stamp_ledger INSERT + stamp_tier UPSERT + 등급업 여부 반환.
merchant_id qr_nonce client_lat client_lng
→ 200 success · cooldown · invalid_nonce · out_of_range
→ body: { stamp_count, tier, tier_upgraded, unlocked_benefits[] }
GET
/v1/me/stamp-cards
지갑 탭의 '스탬프 카드' 세그먼트 진입 시. 사용자의 모든 매장별 등급·스탬프 수·다음 혜택까지 거리 리스트 반환. 감쇠 임박(D-7) 매장 상단 정렬.
GET
/v1/merchants/:id/tier-detail
매장 상세 화면 진입 시 내 등급·해금된 혜택·다음 스탬프 쿨다운 반환. 단골만 보이는 시크릿 쿠폰은 이 응답에서만 노출.
POST
/v1/merchants/:id/qr-nonce 사장님 앱

/v1/merchants/:id/regulars 사장님 앱

리스크
& 완화

적립 앱의 두 가지 고전적 함정 — 어뷰징과 체리피킹 손님. 시작 단계에서 방어 설계.

!
Risk 01 · Abuse
조직적 QR 크롤러 — 1인당 100매장 허위 적립
동일 기기가 하루에 수십 매장을 이동 없이 스캔하는 패턴. GPS spoofing 앱 사용 가능성.
완화
하루 최대 5매장 적립 상한. 초과 시 추가 인증(결제 연동) 요구. Android는 Mock Location 탐지.
!
Risk 02 · Merchant Quality
사장님이 시크릿 혜택을 매력 없이 설정
"5% 할인" 같은 무성의한 시크릿 쿠폰으로 단골 승급 기대감 자체 붕괴.
완화
사장님 앱에 추천 혜택 템플릿 제공 ("히든 메뉴 무료 업그레이드" 등). 시크릿 쿠폰은 일반 대비 ≥ 1.5배 할인율 가이드라인 noti.
!
Risk 03 · Churn
감쇠 시스템이 오히려 이탈 가속
등급 강등 경고 푸시가 부정적으로 받아들여져 앱 삭제 트리거가 될 수 있다.
완화
감쇠 Push는 부정어 대신 "오랜만에 얼굴 보여주세요" 톤. 강등 후에도 스탬프 수는 보존 (tier만 내림). 재방문 1회로 복구.
!
Risk 04 · Privacy
사장님 대시보드의 단골 리스트가 감시감 유발
"○○님 3번째 방문" 같은 표시가 손님에게 부담. 사장님이 사적으로 파악해도 앱이 대놓고 알려주는 건 다름.
완화
사장님에게 보이는 건 닉네임·등급·총 방문수만. 개별 방문일시·동선 절대 노출 안함. 사용자 설정에서 '단골 정보 공유' OFF 가능.
!
Risk 05 · Cold Start
단골 되기까지 너무 멀어 보임 (10장)
초기 사용자가 1/10 단계에서 동기 상실. 진전감 부재.
완화
첫 스캔 직후 환영 쿠폰 즉시 증정 + 5장(regular) 중간 등급에서도 작은 혜택. "단골까지 7번 남음" 구체적 잔여 수 표시.

오픈 퀘스천

PM·디자인·엔지니어가 스펙 확정 전 함께 답해야 할 질문들.

?
Q1 · Product
등급 체계는 3단(new·regular·vip) vs. 5단(bronze·silver·gold·platinum·diamond)?
5단은 세분화된 성취감 주지만 복잡도 증가. 동네 카페 스케일에 5단은 과할 수도.
결정 기준
MVP는 3단. 6개월 후 VIP 도달률이 20% 넘으면 5단 확장 검토.
?
Q2 · UX
스캔 방식 — 사용자가 매장 QR 스캔 vs. 매장이 사용자 QR 스캔?
전자는 마찰 최소(사용자 단독 완결). 후자는 사장님 인지·검증 강화. Starbucks는 후자, 종이 스탬프 카드는 전자에 가까움.
결정 기준
MVP는 전자(사용자 → 매장 QR). 파일럿 결과 사장님 니즈 강하면 Phase 2에 양방향 지원.
?
Q3 · Design
스탬프 카드 디자인 — 매장별 커스터마이징 vs. 통일 템플릿?
매장별 커스터마이징은 브랜드감 ↑, 디자인 불균형 위험. 통일 템플릿은 심미성 보장하지만 개성 ↓.
결정 기준
통일 그리드 + 매장 컬러 1개·로고 1개만 주입 가능. 중도.
?
Q4 · Business
사장님 유료 기능으로 단골 리스트·인사이트 분리?
MVP는 전체 무료. 장기적으로 단골 대시보드·타겟 마케팅은 수익 모델 후보.
결정 기준
Phase 1 전부 무료. Phase 2부터 'Pro 대시보드' (단골 타겟 푸시, 이탈 예측) 월 구독 모델 A/B.
