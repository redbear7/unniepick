01

화면 구성 · 4개 블록

Block A
프로필 헤더
닉네임 이니셜 아바타(64px · 오렌지 그라데이션), 닉네임 + 레벨 뱃지(브론즈/실버/골드), 가입일(joinedDays), 우측 "프로필 수정" 버튼.
하단 레벨 프로그레스바(0~100%) + 다음 레벨까지 남은 포인트.
최하단 3분할 카운트: 보유 쿠폰 · 팔로잉 · 사용 완료.
Block B
긴급 쿠폰 리마인더
"오늘 안에 써야하는 쿠폰 N개" 카드. 오렌지 라이트 그라데이션 배경 + 오렌지 보더. ⏰ 이모지 + 가게명 리스트. 우측 › 쉐브론 → 쿠폰함 탭으로 이동. 당일 만료 쿠폰이 0개면 렌더하지 않음.
Block C
메뉴 그룹 3종
활동(4개) · 혜택(3개) · 계정(5개). 각 그룹은 섹션 라벨 + 흰 카드 그룹. 행마다 아이콘(32px · 그레이 배경) · 라벨 · 서브텍스트 · 배지(옵션) · › 쉐브론.
배지는 긴급 숫자(빨강) 또는 "NEW"(빨강).
Block D
하단 진입 + 계정 액션
"가게 사장님이신가요? 사장님 앱 다운로드 ›" 카드 (AppStore 딥링크). 그 아래 로그아웃 · 회원탈퇴 (텍스트 버튼), 최하단 버전 표기 (UnniePick v1.0.0).

03

데이터 소스 · Supabase 연동

Query A
profiles 테이블
select nickname, joined_at, level, level_progress, location_name from profiles where id = auth.uid()
joinedDays는 클라이언트에서 differenceInDays(now, joined_at)로 계산.
Query B
카운트 3종
단일 RPC my_summary_counts():
보유 쿠폰 = user_coupons where status='active',
팔로잉 = follows where user_id=auth.uid(),
사용 완료 = user_coupons where status='used'.
Query C
긴급 쿠폰 리마인더
select store_name, count(*) from user_coupons where expires_at < today_23:59:59 and status='active' group by store_name. 결과 0개면 Block B 렌더 스킵.
Caching
React Query 5분 TTL
queryKey: ['my', userId], staleTime: 5 * 60 * 1000. 쿠폰 저장/사용 뮤테이션 성공 시 invalidate.

06

엣지 케이스 · 7종

1. 닉네임 미설정
회원가입 S3를 건너뛴 계정은 없어야 하지만 fallback 으로 "언니님"으로 표시. 이니셜은 "언".
2. 레벨 0% / 100%
0%는 프로그레스바 빈 상태 + "브론즈 시작". 100%는 "실버 승급 대기 중" 으로 텍스트 전환.
3. 카운트 0
보유 쿠폰 0 → 숫자 그대로 "0" 표시. 클릭 시 쿠폰함 빈 상태로 이동.
4. 위치 미설정
GPS 거부 + 수동 설정도 안 한 경우 "위치 설정 필요" 오렌지 텍스트 + 설정 화면 딥링크.
5. 생일 준비중
생일이 당월이 아니면 "{N}월 생일 · 준비중" · disabled. 당월이면 "생일 쿠폰 받기" 오렌지 배지.
6. 친구 초대 받은 후
이미 사용한 사용자는 "NEW" 제거, 서브텍스트 "초대 3회 · 누적 9,000원" 로 변경.
7. 로그아웃 · 탈퇴 확인
둘 다 iOS 네이티브 Alert 로 2단계 확인. 탈퇴는 "진짜 탈퇴하시겠어요? 쿠폰 12장이 삭제됩니다" 강조.
