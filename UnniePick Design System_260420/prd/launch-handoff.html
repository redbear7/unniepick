01

범위 정의
& 제외 항목

Phase 1은 사용자 앱 iOS에서 가장 ICE 높은 2개 루프만 집중 런칭. 사장님 도구와 월드컵 수익 루프는 별도 트랙.

In Scope
Spec 01 + Spec 02
Spec 01 지갑 자동 발견 — iOS Live Activity + Widget으로 매장 진입 시 쿠폰 리마인드.
Spec 02 스탬프 × 단골 등급 — QR 스캔으로 도장을 모으고 10개 완성 시 단골 해금 + 시크릿 쿠폰.
두 루프는 coupons 테이블 · GPS 인프라 · Live Activity 파이프라인을 공유.
Out of scope — Phase 2 이후
Spec 03 월드컵 딜 소싱 · 수익 루프 검증 후 Phase 2
사장님 앱 대시보드 · 별도 팀 · 별도 트랙
Android · iOS 첫 런칭 후
쿠폰 발행 툴 · Seed 데이터는 수동 CSV Import
Phase 1 런칭 후 4주간 지표를 수집한 뒤 Phase 2 착수 여부를 재평가. 사장님 앱은 별도 PM이 병렬로 진행.
02

기술 스택
& 인프라

기존 언니픽 레포를 기반으로 iOS 네이티브 모듈 추가. Swift/SwiftUI 중심 + Supabase + 기존 React Native 앱 위에 네이티브 브릿지.

📱 iOS — 신규 네이티브 레이어
Live Activity · Widget: ActivityKit, WidgetKit — iOS 16.2+
위치 감지: CLLocationManager · 배터리 최적화 significantLocationChanges + CLRegion geofencing
QR 스캐너: AVFoundation AVCaptureMetadataOutput · 오프라인 검증
Push: APNs + Supabase Edge Functions 트리거
RN 브릿지: NativeModule 2개 — UPGeofence, UPLiveActivity
🗄 Backend — 기존 스키마 확장
DB: Supabase Postgres + PostGIS 확장 (매장 좌표 · 반경 쿼리)
신규 테이블: stamp_cards, stamp_events, secret_coupons
기존 확장: coupons 에 is_secret, unlock_rule 컬럼
Edge Functions: scan-stamp, trigger-geofence, unlock-regular
RLS: 사용자 본인 데이터만 조회 · 사장님은 is_owner_of
🧪 QA · 배포
CI: GitHub Actions — fastlane beta 자동 업로드
TestFlight: 내부 10명 → 클로즈드 30명
지표: Amplitude + Sentry iOS
지역 한정 런칭: 창원시 의창구·성산구 2개 구 (반경 약 5km)
Seed 매장: 카페 23개 + 음식점 18개 + 미용 12개 — CSV Import
🎨 디자인 · 자산
디자인 시스템: UnniePick iOS UI Kit.html + colors_and_type.css Wanted Sans · Orange #FF6F0F
홈 베리에이션: V1(표준) / V2(발견) / V3(소셜) — 상단 탭으로 A/B 테스트
프로토타입 레퍼런스: prototype/spec-01, prototype/spec-02
아이콘·일러스트: 기존 언니픽 브랜드 팩 재사용
Figma 핸드오프: 필요시 HTML 프로토타입을 Figma Dev Mode로 변환
