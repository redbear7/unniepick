PART 01 · HOME FEED STRUCTURE
🏠
HOME FEED
홈 피드 3-Zone 구조

쿠폰이 많아질수록 "무엇을 위에 보여줄 것인가"가 핵심이다. 피드를 3개 존(Zone)으로 나눠 각기 다른 목적으로 운영한다.

홈 피드 레이아웃 순서
Z1
🔥 긴급 ZONE — "지금 이 순간"
오늘 마감 · 잔여 10장 이하 · 발행 3시간 이내 · 내가 팔로우한 가게 우선
목적: 즉각 행동 유발 (FOMO)
↓
Z2
⭐ 팔로우 ZONE — "내 가게들 소식"
팔로우한 가게의 모든 쿠폰 · 시간 역순 · 이미 저장한 쿠폰은 아래로
목적: 팔로우 관계 유지, 쿠폰 저장
↓
Z3
✨ 추천 ZONE — "이런 가게 어때요?"
팔로우 안 한 가게 · 취향 기반 추천 · 내 주변 인기 쿠폰
목적: 신규 가게 발견, 팔로우 유도
설계 원칙
팔로우 가게가 없을 때는 Z3(추천)만 보인다 → 팔로우를 유도.
팔로우 가게가 많아지면 Z2가 피드의 대부분을 채운다 → 팔로우가 가치를 만든다.
Z1은 언제나 최상단 고정 — 놓치면 안 되는 쿠폰이 항상 보인다.

PART 02 · COUPON RANKING ALGORITHM
📊
SCORING SYSTEM
쿠폰 노출 스코어링 공식

쿠폰이 수백 개 쌓였을 때 무엇을 위에 보여줄 것인가. 5가지 요소를 가중치 합산해 스코어를 계산한다.

FEED SCORE FORMULA
Score = U × 0.35 + F × 0.25 + T × 0.20 + Q × 0.12 + D × 0.08
U = 취향 적합도 F = 팔로우 여부 T = 시간 긴박도 Q = 품질 신호 D = 거리
요소 변수명 가중치 계산 방법 최고점 조건
취향 적합도 U 35% 사용자 행동 기반 카테고리 선호도 × 쿠폰 카테고리 일치율 저장/사용 이력과 카테고리 완전 일치
팔로우 여부 F 25% 팔로우 가게 = 1.0 / 미팔로우 = 0.3 / 차단 = 0 팔로우한 가게의 쿠폰
시간 긴박도 T 20% 발행 후 경과 시간 반비례 + 만료까지 남은 시간 반비례 방금 발행 + 오늘 마감
품질 신호 Q 12% (저장률 + 사용률 + 재발행 횟수) / 3 저장률 30%+ 쿠폰
거리 D 8% 1 - (거리 / 최대 반경). 가까울수록 높음 100m 이내
왜 취향 적합도(U)가 가장 높은가?
팔로우 여부(F)만 높이면 팔로우를 많이 한 사람은 피드가 너무 많아지고, 처음 온 사람은 피드가 비어버린다. 취향 학습이 잘 될수록 팔로우 안 한 가게의 쿠폰도 적절히 보여줄 수 있어 신규 발견(Discovery)이 가능해진다.
실제 계산 예시 — 같은 시간에 3개 쿠폰이 올라왔을 때
쿠폰UFTQD최종 스코어순위
카페봄날 · 아메리카노 1+1
팔로우 · 카페 자주 씀 · 방금 · 오늘 마감		 0.9 1.0 0.95 0.7 0.9 0.918 🥇 1위
미용실언니 · 컷트 20%
팔로우 · 미용 가끔 · 2시간 전 · D-3		 0.6 1.0 0.6 0.8 0.7 0.726 🥈 2위
네일살롱 핑크 · 젤네일 30%
미팔로우 · 네일 자주 씀 · 오늘 · D-5		 0.85 0.3 0.7 0.6 0.5 0.614 🥉 3위
INSIGHT
네일살롱은 팔로우를 안 했지만 취향 적합도(U=0.85)가 높아 3위로 피드에 노출된다. 이것이 "발견(Discovery)" 기능 — 팔로우 안 해도 좋아할 만한 가게를 보여준다. 미팔로우 가게가 노출되면 팔로우 버튼을 누를 확률이 높아진다 → 팔로우 성장.
⏱
URGENCY LOGIC
시간 긴박도(T) 상세 계산

시간 긴박도는 두 가지 시간을 동시에 고려한다. 발행 후 얼마나 지났는가 + 만료까지 얼마나 남았는가.

⬆️ 신선도 (Freshness)

발행 후 시간이 지날수록 점수 감소.
발행 직후(0~3h) → 1.0
3~12시간 → 0.7
12~24시간 → 0.5
1~3일 → 0.3
3일 이후 → 0.1

⬇️ 만료 긴박도 (Expiry Urgency)

만료가 가까울수록 점수 증가.
오늘 마감 → 1.0 (최우선)
D-1 → 0.9
D-2~3 → 0.7
D-4~7 → 0.5
1주일 초과 → 0.3

특수 상황T 점수 처리이유
잔여 수량 20% 이하T × 1.5 (최대 1.0)FOMO 극대화 → 즉시 저장 유도
이미 저장한 쿠폰T × 0.1이미 저장했으면 다시 보여줄 필요 없음
사용 완료한 쿠폰T = 0 (피드 제외)완전히 숨김
사장님 재알림 발송T 초기화 (방금으로 리셋)재알림 = 새 쿠폰처럼 다시 노출
PART 03 · USER PREFERENCE SYSTEM
🧠
PREFERENCE LEARNING
사용자 취향 — 수집 · 학습 · 반영

취향을 묻지 않고 행동에서 읽는다. 사용자가 무엇을 탭하고, 저장하고, 사용했는지가 가장 정직한 취향 신호다.

① 행동 시그널 — 가중치별 취향 신호
행동취향 점수학습 내용이유
🎟 쿠폰 사용 +10점 카테고리, 가게, 할인 유형, 시간대 가장 강력한 신호. 실제 방문까지 했다.
💾 쿠폰 저장 +5점 카테고리, 가게, 할인율 범위 관심 있다는 명시적 신호.
➕ 가게 팔로우 +5점 카테고리, 위치, 가게 규모 지속적인 관심 표현.
👁️ 가게 프로필 조회 +2점 카테고리, 위치 관심이 있어 클릭했다.
⏱ 카드 3초 이상 읽기 +1점 카테고리, 쿠폰 유형 스크롤 멈춤 = 관심 신호.
🔄 카테고리 탭 선택 +1점 선택한 카테고리 명시적 카테고리 선호.
⬇️ 카드 빠르게 스크롤 -1점 해당 카테고리 관심도 감소 관심 없다는 암묵적 신호.
🚫 가게 언팔로우 -3점 카테고리, 해당 가게 명시적 비선호 신호.

취향을 피드에 반영하는 4가지 방법
1
취향 카테고리 피드 탭 자동 생성

카테고리 점수 40점 이상이면 피드 상단 탭에 해당 카테고리가 자동으로 추가된다. 카페를 자주 쓰면 "☕카페" 탭이 생기고, 그 탭을 누르면 카페 쿠폰만 모아서 본다.

동작 예시
홍길동: 카페(88) · 네일(72) 탭 자동 생성
피드 탭: [전체] [팔로우] [☕카페 ★] [💅네일 ★] [🍽음식] [✂️미용]
★ 표시 = 내 취향 탭 (가장 자주 클릭하게 됨)
2
시간대별 맞춤 알림

사용자가 주로 앱을 여는 시간에 맞춰 푸시 알림 시간을 자동 최적화한다.

상황알림 내용발송 시간
점심 시간대 활성 사용자"오늘 점심 근처 쿠폰이 있어요 🍱"오전 11:30
카페 자주 쓰는 사용자"팔로우 가게에서 카페 쿠폰 발행!"오전 10:00
주말 활동적 사용자"주말 특가 쿠폰이 쌓여 있어요 🎟"금요일 저녁
마감 임박 저장 쿠폰 있을 때"저장한 쿠폰이 내일 만료돼요 ⏰"D-1 저녁 7시
3
비슷한 가게 추천 (Z3 구역)

카페봄날을 3번 방문했다면, 비슷한 분위기의 다른 카페를 추천 존에 노출한다. 팔로우 확장을 유도한다.

유사도 계산 기준

• 같은 카테고리
• 비슷한 할인율 범위
• 비슷한 거리대
• 이미 팔로우한 가게와 동시 팔로우 비율
• 비슷한 팔로워 수 (규모)

추천 카드 UI

피드 Z3에 "취향추천" 보라색 배지로 표시.
"카페봄날을 좋아하신다면" 문구 + 비슷한 가게.
1주일에 최대 3개 추천 가게 (과하지 않게).

4
명시적 취향 설정 (온보딩 + 설정)

처음 가입 시와 설정 화면에서 직접 선택할 수 있는 취향 옵션을 제공한다. 행동 데이터가 없는 신규 사용자에게 특히 중요하다.

☕ 카페 🍽 음식 💅 네일 ✂️ 미용 🎂 베이커리 🍷 술집 💆 마사지 🏋️ 헬스
온보딩 취향 선택 → 초기 U 점수에 반영 (카테고리별 +50점 기본 부여)
이후 실제 행동으로 점수가 조정되면서 더 정확해진다.
설정 → 취향 관리에서 언제든 수정 가능.
PART 04 · IMPLEMENTATION
🛢
SUPABASE SCHEMA
취향 학습 DB 스키마
-- 사용자 취향 점수 테이블 CREATE TABLE user_preferences ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID REFERENCES auth.users(id), category TEXT NOT NULL, -- 'cafe' | 'food' | 'beauty' | 'nail' score INTEGER DEFAULT 0, -- 누적 취향 점수 updated_at TIMESTAMP DEFAULT NOW(), UNIQUE(user_id, category) ); -- 사용자 행동 로그 (취향 학습 원데이터) CREATE TABLE user_events ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID REFERENCES auth.users(id), event_type TEXT NOT NULL, -- 'view' | 'save' | 'use' | 'follow' | 'scroll_past' coupon_id UUID REFERENCES coupons(id), store_id UUID REFERENCES stores(id), category TEXT, score_delta INTEGER, -- +10, +5, +2, +1, -1, -3 created_at TIMESTAMP DEFAULT NOW() ); -- 피드 스코어 캐시 (매 5분 갱신) CREATE TABLE feed_scores ( user_id UUID REFERENCES auth.users(id), coupon_id UUID REFERENCES coupons(id), score FLOAT, -- 최종 스코어 (0~1) u_score FLOAT, -- 취향 적합도 f_score FLOAT, -- 팔로우 여부 t_score FLOAT, -- 시간 긴박도 is_followed BOOLEAN, calculated_at TIMESTAMP DEFAULT NOW(), PRIMARY KEY(user_id, coupon_id) );
🔄 점수 업데이트 흐름

사용자 행동 발생
→ user_events에 INSERT
→ Supabase Edge Function 트리거
→ user_preferences.score UPDATE
→ feed_scores 재계산 (5분 배치)
→ 다음 피드 요청 시 반영

⚡ 피드 쿼리 최적화

feed_scores 테이블에 미리 계산된 점수 캐시.
피드 요청 시 SELECT + ORDER BY score DESC.
실시간 계산 없이 빠른 응답.
쿠폰 발행/만료 시 해당 쿠폰 스코어만 갱신.
